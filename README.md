# Work Time Logger (wtl)

Work Time Logger は、TUIベースの作業時間記録ツールです。
日々の業務時間を記録し、柔軟な形式でレポートを出力できます。

## 🌟 主な機能

### 📦 ターミナルUI (`wtlui`)

キーボードだけで操作が完結する、生産性の高いダッシュボードです。

- **ファジー検索 (`s` キー)**: ジョブ名やコードから瞬時に目的のタスクを見つけて、すぐに計測を開始できます。
- **インプレース編集**: 記録済みのログは、表の上で直接修正可能。開始・終了時刻は `↑` `↓` キーで手軽に微調整できます。
- **ダッシュボード (`d` キー)**: 今日の作業状況や累計時間をひと目で把握。
- **柔軟なフィルタリング (`f` キー)**: プロジェクトや日付でログを絞り込み、必要な情報だけを表示できます。

### ⌨️ CLI (`wtl`)

自動化やスクリプト、素早い操作に適したコマンド群です。

- **一括インポート**: CSVファイルから大量のジョブデータを一括で登録可能。
- **柔軟なエクスポート**: TOML形式のプロファイルを使用して、Jinja2テンプレートによる高度にカスタマイズされたCSVレポートを出力。
- **シェル補完**: Typerによる強力な自動補完で、プロジェクトやジョブの入力をサポート。
- **未割り当てタイマー**: とりあえずタイマーを走らせておき、後からジョブを割り当てることも可能です。

各コマンドの詳細は [CLI Reference](docs/cli-reference.md) を参照してください。

## 🚀 クイックスタート

まずはプロジェクトを作成し、記録を始めてみましょう。

```bash
# 1. 新しいプロジェクトを追加
wtl project add -n "Example Project"

# 2. TUIを起動して作業をトラッキング
wtlui
```

> TUI内で操作に迷ったら `h`（または `F1`）キーを押してヘルプを表示してください。

## 🛠️ カスタマイズ (`profile.toml`)

`~/.wtl/profile.toml` を作成（または編集）することで、ツールの挙動をあなた専用にカスタマイズできます。

- **高度なインポート**: `[import.mapping]` を使い、CSVの複数カラムを組み合わせてジョブコードを自動生成。
- **インテリジェントな集計**: `[export.extract]` で正規表現を使い、ジョブコードからプロジェクト種別やチケット番号を抽出して集計。
- **自由なレポート形式**: Jinja2 テンプレートを使用し、社内の勤怠管理システムやクライアント提出用など、あらゆるCSVフォーマットに合わせたエクスポートが可能です。

詳細は [docs/profile.md](docs/profile.md) を参照してください。
